## 언어 설정
- **모든 응답은 한국어로** 작성
- 주석은 한국어로 작성
- 변수명, 함수명은 영어 (camelCase)

## 커밋 메시지
- 한국어로 작성
- 형식: `타입: 설명`
- 예시: `기능: 로그인 페이지 추가`, `수정: 버튼 클릭 오류 해결`

## 코딩 컨벤션
- 들여쓰기: 2칸 스페이스
- 문자열: 작은따옴표(`'`) 사용
- 세미콜론: 생략

## 금지 사항
- console.log 남기지 않기 (디버깅 후 삭제)
- 주석 없는 복잡한 로직 금지
- 하드코딩된 값 금지 (상수로 분리)

## 파일 구조
```
src/
├── components/   # 재사용 컴포넌트
├── pages/        # 페이지 컴포넌트
├── styles/       # CSS 파일
└── utils/        # 유틸리티 함수
```

## 협업 규칙

### PR 규칙
- 제목: `[타입] 설명` (예: `[기능] 로그인 추가`)
- 본문: 변경 사항, 테스트 방법 필수
- 리뷰어 최소 1명 승인 필요

### 브랜치 전략
- `main`: 배포용 (직접 푸시 금지)
- `develop`: 개발 통합
- `feature/*`: 기능 개발
- `fix/*`: 버그 수정
