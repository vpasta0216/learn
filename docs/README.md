# 📁 프로젝트 문서

> PDCA 방법론 기반 문서 관리

---

## 폴더 구조

```
docs/
├── 01-plan/       # Plan: 계획 문서
├── 02-design/     # Do: 설계 문서
├── 03-analysis/   # Check: 분석 문서
└── 04-report/     # Act: 보고서
```

---

## PDCA 사이클

```
Plan (계획)
  │
  ├─→ 요구사항 정의
  ├─→ 목표 설정
  └─→ 범위 확정
       │
       ▼
Do (실행)
  │
  ├─→ 설계서 작성
  ├─→ 코드 구현
  └─→ 테스트 작성
       │
       ▼
Check (검증)
  │
  ├─→ 설계 vs 구현 비교
  ├─→ 코드 품질 분석
  └─→ 테스트 결과 확인
       │
       ▼
Act (개선)
  │
  ├─→ 회고 작성
  ├─→ 학습 내용 정리
  └─→ 다음 사이클 계획
       │
       ▼
  (다시 Plan으로...)
```

---

## 파일 명명 규칙

```
[타입]-[번호]-[기능명].md

예시:
- PLAN-001-로그인기능.md
- DESIGN-001-로그인기능.md
- ANALYSIS-001-로그인기능.md
- REPORT-001-로그인기능.md
```

---

## 문서 작성 순서

1. **Plan 작성** → 리뷰 → 승인
2. **Design 작성** → 리뷰 → 승인
3. **코드 구현**
4. **Analysis 작성** (설계 vs 구현 비교)
5. **Report 작성** (회고 및 개선점)
6. **CLAUDE.md 업데이트** (배운 규칙 추가)
